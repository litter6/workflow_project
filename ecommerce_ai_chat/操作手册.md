## 1. 系统概览

本工具是一款面向电商运营人员的 AI 内容生成平台，核心能力如下：

| 能力 | 说明 |
|------|------|
| 文案生成 | 输入产品描述（可附参考图/视频）→ AI 深度分析 → 生成完整电商文案 |
| 图片生成 | 基于分析结果自动写英文提示词 → gpt-image-2 生成产品图 |
| 图片融合 | 将 AI 生成图与用户上传参考图融合，保留产品主体 + 参考风格 |
| 视频生成 | 基于分析结果生成 Sora 视频提示词 → Sora-2 生成 5 秒广告视频 |
| 视频处理 | 上传已有视频 → Whisper 语音转录 → AI 生成营销文案 → MoviePy 合成电商新视频（含字幕、主标题、卖点、CTA） |
| 多平台适配 | 内置淘宝、京东、拼多多、抖音、小红书、微信私域、亚马逊七大平台专属配置 |
| 多轮对话 | 保留上下文，AI 自动理解"上面"、"之前"等指代，持续优化内容 |

## 2. 快速启动

### 后端

```powershell
cd ecommerce_ai_chat\backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8001
```

后端运行于 `http://localhost:8001`，首次启动自动创建 SQLite 数据库 `ecommerce_chat.db`。

> **前置条件**：Clash 代理需运行在 `http://127.0.0.1:7890`（OpenAI 所有调用都经此代理）。

### 前端

```powershell
cd ecommerce_ai_chat\frontend
npm install
npm run dev
```

前端运行于 `http://localhost:5173`，Vite 自动将 `/api` 代理到后端。

## 5. 功能一：自由输入模式

### 5.1 纯文字请求

1. 在输入框输入需求，如：`帮我写一双运动鞋的详情页`
2. 选择生成类型（**文案 / 图片 / 视频**）和模型（**ChatGPT / DeepSeek V4**）
3. 按 `Enter` 或点击发送按钮

**系统处理流程：**

```
用户输入
  ↓
/api/analyze（GPT-4o）：提取关键词、产品类型、目标客群、核心卖点、情感触发点等
  ↓ 分析结果气泡展示
/api/generate：根据类型生成文案 / 图片 / 视频
  ↓ 结果展示
```

### 5.2 附带参考图

- **拖拽**：将图片直接拖入输入框（支持多张）
- **点击上传**：点击输入框左侧的 📎 按钮选择文件
- 支持格式：`image/*`（JPG / PNG / WEBP 等）

> AI 会先深度分析图片的构图、色调、光线、背景等视觉特征，再将这些特征融入生成内容，实现"同款风格"复现。

### 5.3 附带参考视频

在自由输入模式下可上传视频文件，AI 会均匀抽取最多 16 帧进行视觉分析，分析结果影响文案和图片提示词的生成。

> 注意：此处视频仅用于"分析参考"，不做字幕合成处理。如需完整视频处理请使用「视频处理」Tab。

### 5.4 生成结果操作

分析气泡右上角提供以下操作：

| 按钮 | 功能 |
|------|------|
| ↻ 重新生成 | 用相同分析结果重新调用生成接口，适合结果不满意时刷新 |
| 上传新参考图重新分析 | 拖入/选择新图片，配合补充说明重新触发完整分析 + 生成流程 |
| 🔀 合并生成 | **仅当已有生成图片时出现**，将当前 AI 生成图与新上传图片融合，保留产品主体并融入新图风格 |

### 5.5 多轮对话

系统自动记录每轮对话的上下文，下一轮发送时会附带历史，AI 能理解"和上面一样的风格"、"把之前的文案改得更活泼"等指代表达。

## 6. 功能二：标准配置模式（PromptBuilder）

适用于需要精确指定创作条件的场景。

### 必填字段

| 字段 | 说明 |
|------|------|
| 目标平台 * | 从 7 个平台中选一个，后续选项随平台变化 |
| 产品类型 * | 从 10 个品类中选一个 |
| 内容类型 * | 平台专属内容形式，如"短视频脚本（15秒）"、"Listing 标题"等 |

### 选填字段

| 字段 | 说明 |
|------|------|
| 目标受众 | 细分用户画像 |
| 结果风格 | 如"接地气口语化"、"严谨技术流" |
| 平台专属扩展 | 每个平台有 3 个专属配置项，如抖音的"黄金3秒钩子"、亚马逊的"目标站点" |
| 补充说明 | 品牌名、关键词、竞品参考等自由文本 |

### 支持平台

| 平台 | 典型内容类型 |
|------|------------|
| 淘宝/天猫 | 主图文案、详情页卖点、促销文案 |
| 京东 | 商品标题、五大卖点、参数规格说明 |
| 拼多多 | 爆款标题、拼团文案、百亿补贴文案 |
| 抖音 | 短视频脚本、直播带货话术、达人种草脚本 |
| 小红书 | 种草笔记、测评分享、封面标题 |
| 微信私域 | 朋友圈文案、社群通知、复购激励文案 |
| 亚马逊 | Listing 标题、五点描述、A+ 品牌内容 |

配置完成后点击「生成提示词 →」，系统自动切换到自由输入模式并发送结构化提示词。

## 7. 功能三：视频处理流水线

### 完整流程图

```
① 上传视频（≤200MB，MP4/MOV/AVI）
      ↓
② 点击「分析视频内容，提取关键词」
      ↓ 约 30-60 秒（抽帧 + GPT-4o 视觉分析）
③ 查看分析结果：产品类型、核心卖点、目标客群
   编辑关键词（增加/删除）
   填写附加指令（可选）
   右侧 Remotion 实时预览效果
      ↓
④ 点击「生成电商新视频」
      ↓ 约 2-5 分钟（Whisper 转录 + GPT 生成文案 + MoviePy 合成）
        进度步骤显示：语音转录 → 生成文案 → 合成视频 → 完成
      ↓
⑤ 查看结果：
   - 预览/下载电商新视频（含主标题 + 卖点字幕 + CTA + 语音字幕）
   - 复制营销文案（标题 + 文案 + 卖点 + 话题标签）
   - 下载 SRT 字幕文件
   - 点击「← 重新处理」可调整关键词重新合成
```

### 视频叠加层说明

MoviePy 合成后的新视频包含 4 层文字叠加：

| 层 | 位置 | 时段 | 内容 |
|----|------|------|------|
| 主标题 | 顶部居中，白字深色背景 | 视频前 4 秒或前 30% | AI 生成的产品主标题（8-14字） |
| 卖点文字 | 左右交替，橙色背景 | 视频 20%-75% 均匀分布 | 3-4 条卖点（6-14字/条） |
| 语音字幕 | 底部居中，黑色半透明背景 | 随 Whisper 分段时间轴 | 视频原声语音转录 |
| CTA | 字幕正上方，红色背景 | 结尾 4 秒或后 25% | 行动号召（4-10字） |

### 关键词的作用

关键词同时影响：
- 营销文案（标题/正文/话题标签）中的用词取向
- 视频叠加层的主标题和卖点内容

**建议**：保留 AI 提取的核心关键词，删除不准确的，手动补充产品差异化特点。

### Remotion 实时预览

右侧预览窗口使用 Remotion Player 实时渲染，修改关键词后预览立即更新。这是近似效果预览，最终视频由后端 MoviePy 高质量合成（含真实字幕时间轴）。

## 8. 会话管理

- 每次发送消息前如果没有活跃会话，系统**自动创建**新会话
- 每次发送消息后，内容自动保存（2 秒防抖）
- 会话标题取自第一条用户消息的前 30 个字符
- 切换会话：点击左侧会话列表中的任意一条
- 新建会话：点击「＋ 新建对话」
- 删除会话：点击会话旁的「删除」图标（不可恢复）

> **注意**：保存时大图片（base64 参考图）会自动从消息中剥离，节省存储空间，但不影响历史上下文的分析摘要。

## 9. API 接口速查

所有接口以 `http://localhost:8001` 为基础，需要鉴权的接口在 Header 中传 `Authorization: Bearer <token>`。

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | `/api/auth/register` | 否 | 注册（username + password） |
| POST | `/api/auth/login` | 否 | 登录，返回 token |
| GET  | `/api/sessions` | 是 | 获取会话列表 |
| POST | `/api/sessions` | 是 | 创建新会话 |
| GET  | `/api/sessions/{id}` | 是 | 获取会话详情（含消息列表） |
| PUT  | `/api/sessions/{id}` | 是 | 保存会话（messages + title） |
| DELETE | `/api/sessions/{id}` | 是 | 删除会话 |
| POST | `/api/analyze` | 否 | 分析文本+图片/视频，返回 context |
| POST | `/api/generate` | 否 | 生成文案/图片/视频 |
| POST | `/api/combine` | 否 | 合并两张图片 |
| POST | `/api/video/analyze` | 是 | 视频处理第一阶段：抽帧分析，返回 session_id + 关键词 |
| POST | `/api/video/process` | 是 | 视频处理第二阶段：转录 + 合成，返回新视频 base64 + 文案 |

## 10. 常见问题

### Q1：发送请求后一直转圈不返回

- 检查 Clash 代理是否在 `127.0.0.1:7890` 运行（所有 OpenAI 调用都需要代理）
- 图片生成（gpt-image-2）**约需 100 秒**，请耐心等待
- 视频处理（全流程）**约需 2-5 分钟**，进度条会实时更新

### Q2：图片生成失败，报 500 错误

- 确认 `.env` 中 `OPENAI_API_KEY` 有效且余额充足
- gpt-image-2 对提示词长度有限制，系统已限制在 400 词以内

### Q3：视频分析失败

- 视频文件不能超过 200MB
- 支持格式：MP4、MOV、AVI（需系统安装 ffmpeg，MoviePy 依赖）
- 必须安装 Python 依赖：`moviepy`、`Pillow`

### Q4：视频处理后没有语音字幕

- Whisper 无法识别语音会返回 422 错误："未能从视频中识别到语音内容"
- 检查视频是否有音轨，或音量是否过低

### Q5：关键词编辑后预览没变

- Remotion 预览实时响应关键词变化，但需要等待 React 重渲染（约 0.5 秒）
- 最终合成视频以后端处理结果为准

### Q6：DeepSeek 返回错误

- 确认 `.env` 中 `DEEPSEEK_API_KEY` 有效
- DeepSeek V4 仅用于文案生成，图片/视频生成强制使用 OpenAI

### Q7：登录提示"登录已过期"

- Token 有效期 30 天，过期需重新登录
- 浏览器隐私模式关闭后 localStorage 会被清空

## 12. 异步视频流水线（Celery 版）

> 本节覆盖 2026-05-20 新增的 9 层异步视频处理流水线。与第 7 节的同步流程不同，此版本通过 **Redis + Celery** 在后台异步处理，页面实时轮询进度，支持更长视频和更稳定的处理。

### 12.1 额外环境依赖

| 依赖 | 说明 | 安装 |
|------|------|------|
| Redis | 消息队列 + 结果后端 | `winget install --id Redis.Redis` |
| FFmpeg | 音频提取 / 编码 / BGM 混音 | `winget install --id Gyan.FFmpeg` |

验证：

```powershell
redis-cli ping      # 返回 PONG
ffmpeg -version     # 返回版本信息
```

FFmpeg 安装后若当前 PowerShell 会话找不到命令，刷新 PATH：

```powershell
$env:PATH = [System.Environment]::GetEnvironmentVariable("PATH","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("PATH","User")
```

### 12.2 启动顺序（必须全部启动）

**① 启动 Redis**

```powershell
redis-server
```

**② 启动 Celery Worker**（新开终端，在 `backend/` 目录）

```powershell
cd d:\workflow_project\ecommerce_ai_chat\backend
celery -A celery_app worker --pool=threads --loglevel=info
```

启动日志中必须看到：

```
[tasks]
  . tasks.process_video
```

若没有这行，说明任务模块未加载，视频任务将卡在 `pending`，参见第 12.4 节排查。

**③ 启动 FastAPI**（新开终端）

```powershell
cd d:\workflow_project\ecommerce_ai_chat\backend
uvicorn main:app --reload --port 8001
```

**④ 启动前端**（新开终端）

```powershell
cd d:\workflow_project\ecommerce_ai_chat\frontend
npm run dev
```

### 12.3 使用方式

1. 登录后点击「视频处理」Tab
2. 上传 MP4/MOV/AVI 文件（建议 ≤500MB）
3. 点击「开始处理」，进度条实时更新（每 2 秒轮询）

**9 层流水线阶段：**

| 进度 | 阶段 | 耗时（参考）|
|------|------|------------|
| 0–8% | 视频上传 | 取决于文件大小 |
| 8–22% | FFmpeg 提取音频 | 1–5 秒 |
| 22–40% | Whisper 语音转字幕 | 5–30 秒 |
| 40–60% | GPT-4o 画面分析 | 10–30 秒 |
| 60–70% | GPT-4o 营销文案 | 5–15 秒 |
| 70–80% | MoviePy 字幕烧录 | 10–60 秒 |
| 80–93% | FFmpeg H.264 重编码 | 5–30 秒 |
| 93–98% | （扩展位保留） | — |
| 98–100% | FFmpeg BGM 混音 | 1–5 秒 |

4. 完成后点击「下载视频」获取成品

**可选 BGM：** 将 `backend/assets/bgm.mp3` 放置到该路径，Layer 9 自动混入；文件不存在则跳过。

### 12.4 排查：任务卡在 `pending`

**诊断：**

```powershell
cd d:\workflow_project\ecommerce_ai_chat\backend
python -c "from celery_app import celery_app; print(celery_app.control.inspect().registered())"
```

- 输出中有 `tasks.process_video` → Worker 正常，检查其他原因
- 输出为 `None` 或不含该条目 → Worker 未正确加载任务模块

**解决方案：**

1. 确认 `celery_app.py` 中 `conf.update` 包含 `include=["tasks"]`
2. 杀掉所有旧 Worker 进程（任务管理器 → 结束所有 `celery` 进程）
3. 重新执行步骤 ② 启动 Worker

### 12.5 排查：任务立即变为 `failed`

```powershell
cd d:\workflow_project\ecommerce_ai_chat\backend
python check_jobs.py
```

查看 `error_msg` 列：

| 错误信息 | 原因 | 解决 |
|----------|------|------|
| `ffmpeg not found` | FFmpeg 未在 PATH | 刷新 PATH（见 12.1）|
| `Connection refused` / `APIConnectionError` | Clash 代理未启动 | 启动 Clash，确认 7890 端口 |
| `AuthenticationError` | API Key 无效 | 检查 `.env` 文件 |
| `insufficient_quota` | OpenAI 余额耗尽 | 账户充值 |
| `No such file` | 上传文件路径丢失 | 检查 `UPLOAD_DIR` 磁盘空间 |
